Installing C/C++ Graphics Library in Ubuntu/Fedora ¬

Instructions for Ubuntu / elementaryOS ¬

  1. Make sure you have gcc and g++ compilers installed.
    If not, then install with,
    sudo apt-get install build-essential

  2. Install SDL Libraries
    sudo apt-get install libsdl-image1.2 libsdl-image1.2-dev guile-1.8 guile-1.8-dev libsdl1.2debian libart-2.0-dev libaudiofile-dev libesd0-dev libdirectfb-dev libdirectfb-extra libfreetype6-dev libxext-dev x11proto-xext-dev libfreetype6 libaa1 libaa1-dev libslang2-dev libasound2 libasound2-dev
    If you get errors such as :
    E: Unable to locate package libslang2-dev
    E: Unable to locate package libasound2-dev
    Then execute the following command to update repository list :
    sudo apt-get update
    Run the command to Install SDL libraries again.

  3. Obtain a copy of libgraph-1.0.2.tar.gz by following command
    wget http://ftp.twaren.net/Unix/NonGNU/libgraph/libgraph-1.0.2.tar.gz
    OR if the link doesn't work
    wget http://download.savannah.gnu.org/releases/libgraph/libgraph-1.0.2.tar.gz
    OR download it manually from here or here

  4. Extract libgraph-1.0.2.tar.gz using following command :
    tar xf libgraph-1.0.2.tar.gz
  5. Run the following commands (copy-paste will work, in terminal, paste by right-click menu not by CTRL + V) :
    cd libgraph-1.0.2
    ./configure
    sudo make
    sudo make install
    sudo cp /usr/local/lib/libgraph.* /usr/lib
    If you get
    configure: error: *** SDL version 1.2.0 not found!
    in ./configure
    then SDL libraries were not successfully installed, perform Step 2 again.

  6. Write your C/C++ graphics program, sample program is given below : 

    #include<stdio.h>
#include<graphics.h>

int main()
{
	int GM, GD;
	initgraph(&GM,&GD, NULL);

	circle(320, 240, 60);
	
	scanf("%d",&GD); // to hold the screen
}

  7. Compile your program using :
    g++ myprogram.cpp -lgraph
    OR
    gcc myprogram.c -lgraph

  8. Execute program : ./a.out

Important instructions for writing program ¬

  1. Use NULL in initgraph() function graphics driver path.
  2. Don't forget to add -lgraph argument when compiling.
  3. getch() can't be used in linux operating systems as conio.h header file is not present.
  4. To hold the graphics output screen, use scanf() function instead of getch().

Instructions for Fedora ¬

  1. Obtain a copy of libgraph-1.0.2.tar.gz by following command
    wget http://ftp.twaren.net/Unix/NonGNU/libgraph/libgraph-1.0.2.tar.gz
    OR if the link doesn't work
    wget http://download.savannah.gnu.org/releases/libgraph/libgraph-1.0.2.tar.gz
    OR download it manually from here or here

  2. Extract libgraph-1.0.2.tar.gz using following command :
    tar xf libgraph-1.0.2.tar.gz
  3. Install SDL libraries (run following commands separately)
    yum install SDL.x86_64 SDL-devel.x86_64 SDL_image.x86_64 SDL_image-devel.x86_64

    yum install guile guile.x86_64 guile.i686 guile-devel.x86_64 compat-guile18-devel.x86_64
  4. Run the following commands separately :
    ./configure
    make
    make check
    make install
    sudo cp /usr/local/lib/libgraph.* /usr/lib

  5. Write your C/C++ graphics program, sample program is given below : 

    #include<stdio.h>
#include<graphics.h>

int main()
{
	int GM, GD;
	initgraph(&GM,&GD, NULL);

	circle(320, 240, 60);
	
	scanf("%d",&GD); // to hold the screen
}

  6. Compile your program using :

    g++ myprogram.cpp -lgraph
    OR
    gcc myprogram.c -lgraph

  7. Execute program : ./a.out
